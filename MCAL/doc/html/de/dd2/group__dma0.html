DMAC_RETURN_CODE_t dmacSetDataFormat ( DMA_Stream_TypeDef *  stream,
DMAC_DATA_FORMAT_t  format 
)

Sets the format of the data that shall be transferred. The data format is the same for both the source and the destination.

Parameters
*stream: Pointer to the DMA stream
format: Data format of the data
Returns
DMAC_RETURN_CODE_t
Note
This function shall not be used any longer since it sets the data format (byte, half-word, or word) to the same value.
However, it might be possible that the data format of any peripheral is different from the data format of the memory.
Use dmacSetPeripheralDataFormat() and dmacSetMemoryDataFormat() instead.

This multi-param function has been split up into separate functions. The following table shows the names of the replacement
functions.

SF: Separate functions
Parameter(s) New function
stream + channel dmacSetStreamAndChannel()
source + destination address dmacSetMemoryAddress() and dmacSetPeripheralAddress()
numData dmacSetNumData()
Data format (src and dst) dmacSetDataFormat()
dir dmacSetDataFlowDirection()
irq dmacEnableInterrupt(), dmacEnableInterruptList(), dmacDisableInterrupt(), dmacDisableInterruptList()

Configures the selected DMA stream.

Parameters
*stream: Pointer to the stream
chn: Channel
src: Address of the data source
dest: Address of the data destination
numData: Number of the data
srcFormat: Format of the source data size
dstFormat: Format of the destination data size
dir: Direction of the data transfer
irqType: Interrupt types
Returns
DMAC_RETURN_CODE_t
Note
Before calling this function the given stream must be disabled. After a successful return don't forget to
  • enable the DMA stream
  • add the stream interrupt to NVIC
Although this function uses many parameters it does not support all DMA settings. E.g. this function does not support
FIFO functionality, and it does not let the user select the incrementation mode of memory nor peripherals.
