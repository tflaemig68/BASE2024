void rccAssignClk2MCO ( MCO_CLK_SRC_t  clkSrc,
MCO_TypeDef_t  mco 
)

This function assigns the selected clock to either MCO1 (PA8) or MCO2 (PC9) output.

Parameters
clkSrc: Supported clock types which can be assigned to the MCO
mco: Can be either MCO1 or MCO2
Note
This function takes care about the proper assignment of MCO1/MCO2 to the corresponding GPIO port/pin.
The selected clock to output onto MCO must not exceed 100MHz!

Affected register and bit(s)

Register Bit name Bit(s) Comment
CFGR MCO1/MCO2 22/31 Selects either MCO1 or MCO2 as clock output. Configures the desired GPIO pin internally.

◆ rccSelectPLLClockSource()

void rccSelectPLLClockSource ( PLL_CLOCK_SRC_t  clkSrc)

Selects the clock source which provides the clock to the main PLL and the audio PLL.

Parameters
clkSrc: Clock sources might be PLL_SRC_HSI or PLL_SRC_HSE
Note
This function is different from rccSelectSysclkSrc() in that way that the parameter defines the input to the main PLL circuit! This function is not required when the main PLL circuit is not used, i.e. if HSI or HSE are selected as MCU clock.

Affected register and bit(s)

Register Bit name Bit(s) Comment
PLLCFGR PLLSRC ON 22 0: HSI used as source for PLL and PLLI2S, 1: HSE used for PLL and PLLI2S

◆ rccSelectSysclkSrc()

void rccSelectSysclkSrc ( SYSCLK_SRC_t  sysclkSrc)

This function selects the source of the SYSCLOCK.

Parameters
sysclkSrc: The clock which is used to set the clock of the MCU.
Note
This function is different from rccSelectPLLClockSource() in that way that it controls the MCU clock. It can be either SYSCLKSRC_HSI, SYSCLKSRC_HSE, SYSCLKSRC_PLLP, or SYSCLKSRC_PLLR. Consider that parameters 3 and 4 are the output of the main PLL circuit! To setup the MCU clock (SYSCLK) it is mandatory to use at least this function.

Affected register and bit(s)

Register Bit name Bit(s) Comment
RCC CFGR 1...0 Selects the source of the MCU system clock

◆ rccSetSysclkFreq()

void rccSetSysclkFreq ( uint8_t  fOsc,
uint16_t  mcuFreq 
)

This function calculates the PLLM, PLLN and PLLP settings from the parameters fOsc (oscillator frequency) and mcuFreq (the frequency at which the MCU shall run).

Parameters
fOsc: Frequency of the oscillator used as HSI or HSE
mcuFreq: The frequency which shall be achieved (must be greater than 12)
Note
The maximum frequency which is applicable to the members of the STM32F4 family varies depending on the concrete MCU type. If the value of mcuFreq is higher than the maximum frequency of the selected MCU it is automatically corrected to the maximum value which is specified in the datasheet of the device.

Affected register and bit(s)

Register Bit name Bit(s) Comment
PLLCFGR PLLN 14...6
PLLCFGR PLLM 5...0 Calculates PLLM from the desired clock frequency
PLLCFGR PLLN 14...6 Calculates PLLN from the desired clock frequency
PLLCFGR PLLP 17...16 Calculates PLLP from the desired clock frequency
